# Media Index 0.2.0 开发状态

当前版本：`0.2.0-dev.3`

## 第一性原理结论

这个项目可以实现最初需求，但前提是把它当作“媒体追更决策器”，而不是“搜索到一个链接就转存”的前端工具。

一次自动转存只有在以下事实都成立时才安全：

1. TMDB ID、媒体类型、季和目标集由后端重新确认；
2. 旧 QAS 链接仍有效并包含目标集，或 PanSou 找到更好的有效链接；
3. 分享目录内的文件与 TMDB 集数形成无歧义的一对一映射；
4. `pattern` 精确匹配一个源文件，`replace` 不会产生重名冲突；
5. QAS 只执行已经确认的单项任务，HTTP 触发和实际转存完成分开记录。

## 已完成

- 建立独立的 `develop/0.2.0` 开发线，旧版容器未替换、未重启。
- 后端以 TMDB ID 为唯一事实来源，生成季、集、播出日期和别名目标。
- 旧链接优先；旧数据库没有链接字段时，可从 QAS 历史任务恢复最多 3 个链接依次验证。
- PanSou 改为目标集、中文期数、播出日期、整季逐级放宽查询。
- 修复 PanSou 64 个插件场景下并发过低和无条件 GET/POST 双重超时的问题。
- 候选按标题、季、年份、目标集证据、新鲜度、衍生内容风险排序。
- QAS 分享错误、目录入口和文件列表分别解析，失效链接不会进入重命名。
- 文件级匹配排除预告、花絮、纯享、reaction、OST 等衍生内容。
- 修复 `4K`、`8bit`、`SOME18` 等被误认为集号的问题。
- 每个源文件最多匹配一个 TMDB 集；歧义或低置信度仅作为异常进入待确认。
- 待确认任务复用 QAS 的通知配置，支持确认执行和强制刷新 PanSou 重新搜索。
- 愿望单按 TMDB 日期调度，默认 09:00，并支持每张卡片选择任意整点。
- QAS 执行后通过目标目录文件名对账，未确认任务由后台继续核验。
- 综艺在生成目标时排除加更、纯享、花絮、预告、陪看等衍生 episode，并区分上下期。
- 海报页资源探测也使用 0.2 决策管线，已删除 0.1 的快速假阳性算法和旧 `matching.py`。
- 自动重命名使用锚定且转义的单文件正则，不再用宽泛规则扫整个目录。
- QAS 使用单个内存任务接口执行，不再触发全部任务。
- 追更按 TMDB 播出日期调度，并记录候选、错误、重试和下一次检查时间。

## 2026-07-10 只读实库演练

- 目标：一个现有综艺追更任务的最新已播第 28 集。
- 从旧 QAS 配置恢复：2 个历史链接。
- 旧链接结果：没有可安全复用的目标集资源。
- PanSou：集级查询和整季兜底均正常返回。
- 进入 QAS 验证的候选：5 个。
- 候选结果：分享失效、取消分享或无可读取文件。
- 最终决策：`no_resource`，未执行转存，未修改生产数据库。

这次结果不是搜索算法误判，而是当前候选集合里没有可验证的有效资源。系统选择等待重试，符合“错误自动转存目标为 0”的原则。
