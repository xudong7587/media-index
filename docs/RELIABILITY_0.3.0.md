# MediaIndex 0.3.0 可靠性边界

本版本采用 fail-closed 原则：外部状态无法证明时停止执行，不猜测成功。

- 剧集和综艺转存前必须读取目标目录，只处理最早的下一缺失集。
- QAS、Cookie 或目录查询异常不会退化成“已存 0 集”。
- 规范目录不存在时只允许复用一个确定的历史命名目录；多个兼容目录视为冲突。
- 同一 `TMDB + 类型 + 季 + 保存目标` 只允许一个运行中、待执行或待确认任务。
- QAS 未知返回、周期跳过和“没有新的转存任务”均不能独立证明成功。
- 完成状态要求 QAS 明确成功，且目标目录存在全部预期文件，文件大小大于 0。
- 服务启动时把被重启中断的进程内任务标记为失败并恢复关联追更/愿望单。
- SQLite 启用 WAL、外键、30 秒 busy timeout；登录接口提供失败次数限流。

真实外部服务仍可能改变协议。发生未知响应时任务应停止并保留诊断信息，不应自动扩大匹配范围。
