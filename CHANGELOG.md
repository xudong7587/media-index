# MediaIndex 变更记录

完整发布说明、验证结果和固定镜像标签请查看 [GitHub Releases](https://github.com/xudong7587/media-index/releases)。

## 0.4.7

- 企业微信资源搜索在结果不明确时返回最多 5 个编号候选，不再自动选择第一项。
- 支持通过编号选择资源、待确认任务和资源候选，选择状态按 UserID 保存 30 分钟。
- `/review` 接入既有安全确认与重新匹配流程。
- 设置页拆分为“基础设置”和“通知设置”，并加入内置指令速查。

## 0.4.6

- 企业微信资源任务完成后优先发送带海报的图文消息。
- 海报先缓存到 MediaIndex，再通过公开只读接口提供；不可用时自动回退文字。
- 移除通知独立页面，仅保留右上角小铃铛通知中心。

## 0.4.5

- 企业微信自建应用支持直接发送资源名创建转存任务。
- 资源名默认保存到网盘，`本地 资源名` 保存到本地。
- 网页端和企业微信共用任务入队、去重和安全校验逻辑。

## 0.4.4

- 推送测试返回企业微信或 Telegram 的具体错误信息。
- 错误信息中的 access token 和 Bot Token 自动脱敏。

## 0.4.3

- 新增企业微信自建应用交互回调，支持 Token、EncodingAESKey 和成员白名单。
- 实现回调验证、签名校验、AES 解密、企业 ID 校验和重复投递去重。
- 新增 `/status`、`/review`、`/tracking`、`/wishlist`、`/notifications` 和 `/help`。

## 0.4.2

- 新增企业微信自建应用推送，可按成员、部门和标签指定接收范围。
- 企业微信 access token 按应用缓存。
- 企业微信、群机器人和 Telegram 统一提供测试发送。

## 0.4.1

- 新增 Telegram Bot 和企业微信群机器人推送。
- 转存完成、任务提交、待确认、暂无资源和失败事件可推送到外部渠道。
- 升级前的历史事件不会集中补发。

## 0.4.0

- 新增持久化站内通知中心，支持未读筛选、逐条已读、全部已读和清空。
- 覆盖转存完成、任务提交、需要确认、暂无资源和处理失败。
- 通知事件使用稳定业务键去重。

## 0.3.2

- 支持四位集号及连续范围，并避免把年份误识别为集号。
- 裸数字集号只有在文件序列和候选标题均提供足够证据时才允许自动执行。
- 修复候选正文制造虚假片名命中的问题。

## 0.3.1

- 收紧电影候选标题匹配和日期合辑验证。
- 弱标题、弱文件名及无有效资源的候选不再制造无效待确认任务。
