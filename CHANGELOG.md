# MediaIndex 变更记录

## 0.5.0

- 原生 115 支持 `115cdn.com` 分享链接和查询参数中的提取码，可直接验链与转存。
- 发现页按网盘展示紧凑资源按钮，支持单网盘转存、复制分享链接和一键转存全部有资源网盘。
- 智能追更与愿望单支持切换目标网盘；愿望单增加“立即执行”。
- 智能追更可展开季度集数，按已存状态选择缺失集并手动补齐。
- 设置按夸克与 115 分离保存根目录和分类目录，并支持自定义分类。
- 原生 115 增加独立本地下载目录；网盘暂存目录继续仅用于安全接收、改名和移动。
- 手动补集提示 PanSou 对较早资源的召回限制，愿望单操作区固定网盘按钮宽度并优化日期顺序。

## 0.4.17

- 新增原生 115 Provider：由 MediaIndex 验证分享、匹配文件、转存、改名、移动并核对目标目录，不依赖 MoviePilot 执行转存。
- 设置页支持直接粘贴 115 Cookie，或填写 MoviePilot 地址和 API Key 后从 `P115StrmHelper` 安全导入；配置接口不返回 Cookie。
- 发现页按 `Provider × 季` 并行验证 QAS 与 115，支持单季、最后一季和手动多季批量转存。
- 新增父批次与独立子任务状态；一个网盘失败不会回滚另一个网盘的成功结果，部分成功会单独通知失败原因。
- 数据库补充 Provider、外部任务和批次兼容迁移，旧 QAS 数据继续按原行为运行。
- Bark、Emby 播放联动及 115 扫码/OAuth 已纳入后续路线图，本版本不开放这些未完成能力。

## 0.4.16

- 追更搜索不再使用“喜单”“脱友”等过短别名，优先使用完整标题、发布日期和季信息，减少 PanSou 噪声召回。
- 剧集和综艺候选标题与目标明显无关时直接拒绝，不能再凭短剧目录里的 `12.mp4/13.mp4` 获得集数覆盖分。
- 支持同一期后半段文件，例如“第3期（三）/（四）”按日期连续映射到 TMDB E12/E13。
- 同批目标共享的日期或“第3期”等令牌不再单独用于定集，并且只把真正匹配到文件的集数标记为已转存或已触发。
- 根目录 Compose 增加默认注释的 PanSou 和 QAS 完整部署段，并预填同网络容器地址。
- 新增完整项目使用手册，覆盖首次配置、追更、待确认、更新与备份。

## 0.4.15

- PanSou 候选只包含网盘已经保存的旧集时，反馈“当前已追到可用最新集，无需转存”，不再生成大量无效待确认资源。
- 资源源尚未更新属于正常等待状态，不会因多次重试升级为人工确认；真正存在目标集但文件选择有歧义时仍保留确认流程。
- 资源源尚未更新时固定一小时后复查，手动刷新次数不会把等待时间拉长到四至十二小时。

## 0.4.14

- 支持通过通知设置中的公网地址，让企业微信和 Telegram 读取 MediaIndex 后端缓存海报。
- 手动追更检查结果等直接通知也会立即投递外部渠道，不再只显示在网页通知中心。
- 企业微信自建应用、群机器人和 Telegram 继续优先发送图文消息，失败时回退为文字。

## 0.4.13

- 手动追更会重新搜索已取消的待确认集数，并强制刷新 PanSou 搜索缓存。
- 综艺同一天更新多集时优先加入 `标题 + MMDD` 日期关键词，提高当天新资源命中率。
- 支持把 `同一日期 + 第N期(一)/(二)` 等连续分段安全映射到 TMDB 连续集序号。
- 删除全部待确认候选后，关联追更或愿望单恢复为可重试状态，不再残留为 `needs_review`。
- 重新搜索同一批集数时自动归档旧失败任务并释放幂等键，同时继续阻止运行中或已完成任务重复执行。

## 0.4.12

- 手动立即追更完成后立即刷新并展开通知中心，不再只显示一次加载动画。
- 没有已播新内容、暂未找到资源、网盘检查失败和内部错误均会生成明确的站内结果通知。
- 已创建转存任务时复用转存结果通知，继续反馈已提交、待确认、成功或失败，避免重复通知。

## 0.4.11

- 通知中心卡片优先展示由 MediaIndex 后端缓存的海报，图片不可用时自动回退为原有状态图标。
- 企业微信自建应用、企业微信群机器人和 Telegram 优先发送带海报的图文通知，发送失败时自动回退为文字消息。
- 智能追更和愿望单进入待确认后，可自动向企业微信发送最多 5 个编号候选，成员回复数字即可继续处理。
- 通知设置新增公网访问地址，用于企业微信回调、通知跳转和缓存海报访问。

## 0.4.10

- 修复手动执行追更会绕过未来发布日期的问题；手动执行现在只跳过当天的追更时刻，绝不会提前搜索未来集。
- QAS 已触发任务改为每 10 秒确认一次，智能追更页同步每 10 秒静默刷新，缩短转存完成后的状态回写等待。

完整发布说明、验证结果和固定镜像标签请查看 [GitHub Releases](https://github.com/xudong7587/media-index/releases)。

## 0.4.9

- 智能追更时间改为显式保存，时间输入、保存按钮与任务操作按钮统一对齐。
- 设置页新增吸顶保存栏，可在基础设置和通知设置任意滚动位置保存当前子页。

## 0.4.8

- 智能追更按任务设置的发布时间执行，到点后先刷新 TMDB 剧集信息并扫描网盘已存进度。
- 仅对 TMDB 明确已播出且网盘未保存的新集搜索资源；缺少播出日期时只刷新元数据。
- 智能追更页面新增每个任务独立的追更时间设置。
- 页面右上角新增 GitHub 仓库入口。

## 0.4.7

- 企业微信资源搜索在结果不明确时返回最多 5 个编号候选，不再自动选择第一项。
- 支持通过编号选择资源、待确认任务和资源候选，选择状态按 UserID 保存 30 分钟。
- `/review` 接入既有安全确认与重新匹配流程。
- 设置页拆分为“基础设置”和“通知设置”，并加入内置指令速查。

## 0.4.6

- 企业微信资源任务完成后优先发送带海报的图文消息。
- 海报先缓存到 MediaIndex，再通过公开只读接口提供；不可用时自动回退文字。
- 移除通知独立页面，仅保留右上角小铃铛通知中心。

## 0.4.5

- 企业微信自建应用支持直接发送资源名创建转存任务。
- 资源名默认保存到网盘，`本地 资源名` 保存到本地。
- 网页端和企业微信共用任务入队、去重和安全校验逻辑。

## 0.4.4

- 推送测试返回企业微信或 Telegram 的具体错误信息。
- 错误信息中的 access token 和 Bot Token 自动脱敏。

## 0.4.3

- 新增企业微信自建应用交互回调，支持 Token、EncodingAESKey 和成员白名单。
- 实现回调验证、签名校验、AES 解密、企业 ID 校验和重复投递去重。
- 新增 `/status`、`/review`、`/tracking`、`/wishlist`、`/notifications` 和 `/help`。

## 0.4.2

- 新增企业微信自建应用推送，可按成员、部门和标签指定接收范围。
- 企业微信 access token 按应用缓存。
- 企业微信、群机器人和 Telegram 统一提供测试发送。

## 0.4.1

- 新增 Telegram Bot 和企业微信群机器人推送。
- 转存完成、任务提交、待确认、暂无资源和失败事件可推送到外部渠道。
- 升级前的历史事件不会集中补发。

## 0.4.0

- 新增持久化站内通知中心，支持未读筛选、逐条已读、全部已读和清空。
- 覆盖转存完成、任务提交、需要确认、暂无资源和处理失败。
- 通知事件使用稳定业务键去重。

- 裸数字集号只有在文件序列和候选标题均提供足够证据时才允许自动执行。
- 修复候选正文制造虚假片名命中的问题。

## 0.3.1

- 收紧电影候选标题匹配和日期合辑验证。
- 弱标题、弱文件名及无有效资源的候选不再制造无效待确认任务。
